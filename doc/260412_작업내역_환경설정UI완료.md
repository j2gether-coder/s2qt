## 1. 현재까지 정리된 방향

### 작업 내역

기존의 좌측 목록 / 우측 상세 구조에서 벗어나, **업무형 화면**으로 재구성하는 방향으로 정리되었습니다.

현재 의도한 구조는 다음입니다.

* 상단 안내 카드
* 검색 조건 카드
* 작업 버튼 카드
* 테이블 카드

테이블은 조회용 상세 화면이 아니라 **재작업 대상 선택 화면**으로 보는 방향으로 정리되었습니다.

컬럼은 최소화했습니다.

* 체크박스
* 제목
* 본문 성구
* 저장일
* 연령대

작업 버튼은 표 안이 아니라 **표 밖 상단**으로 두는 방향으로 정리되었습니다.

* 재작업
* 삭제

버튼 규칙도 정리되었습니다.

* 재작업: 1건 선택 시만 가능
* 삭제: 1건 이상 선택 시 가능

즉, 작업 내역은 “상세 보기”보다 “선택 후 다시 작업”에 맞춘 구조입니다.

---

### 환경설정

환경설정은 장년 QT Step 계열의 화면 문법에 맞추는 방향으로 재정리되었습니다.

즉 화면 문법을 아래처럼 맞췄습니다.

* `workspace-step-panel`
* `card`
* `card-plain`
* `step-badge`
* `mini-title`
* `body-note`

상위 탭은 현재 다음과 같이 정리되었습니다.

* 기본 정보
* 부가 기능
* 메뉴 설정
* 안내

각 탭의 역할도 정리되었습니다.

#### 기본 정보

* 기본 이메일
* PIN 설정
* 교회/브랜드 설정

#### 부가 기능

* AI 기능
* SMTP

#### 메뉴 설정

* 사이드 메뉴 표시명
* 사이드 메뉴 표시 여부

#### 안내

* 라이선스
* 도움말
* 사용 가이드

## 2. 환경설정에서 확정된 핵심 UX

### 기본 정보

* 이메일과 PIN은 민감정보 보호 기준으로 사용
* 교회/브랜드 관련 설정은 별도 카드로 유지
* 전체적으로 카드형 구조로 통일

### 부가 기능

* AI 사용 방식은 **한 줄 라디오**

  * 수동
  * 원격
  * 로컬
* SMTP는 입력형을 줄이고, 목적 중심으로 정리
* 발신 이메일은 SMTP에서 직접 수정하지 않고, 기본 정보 이메일을 기준으로 보는 방향

### 메뉴 설정

이 부분이 이번 정리에서 중요합니다.

* 내부 audience 키는 고정

  * `adult`
  * `young_adult`
  * `teen`
  * `child`
* 사용자에게 보이는 메뉴명만 변경 가능
* 표시 여부도 설정 가능

즉,
`중고등부`를 `학생부`처럼 바꿔 쓸 수 있게 하는 방향으로 정리되었습니다.

화면은 카드 안의 **표 방식**으로 가는 것이 더 적절하다고 정리되었습니다.

컬럼은 다음입니다.

* 사용
* 기본 메뉴
* 표시명

저장은 행별 저장보다 **하단 저장 버튼 1개**가 더 적절하다고 정리되었습니다.

## 3. CSS 정리 방향

CSS도 같이 정리되었습니다.

큰 방향은 다음입니다.

* 공통
* 절차형 화면
* 설정별
* 작업내역별
* 보안(PIN)
* 반응형

특히 중요한 점은:

* Settings는 이제 예전 `section-block` 중심보다 **Step 계열 card 구조**가 중심
* History는 예전 좌우 분할 CSS보다 **테이블형 CSS**가 중심
* 버튼 체계는 점진적으로 `button / button-ghost / button-success` 중심으로 정리

또한 환경설정에서
상단 탭 카드와 첫 본문 카드 간 간격 문제도 구조적으로 검토했고,
이건 컨테이너 여백 중복 때문이라는 점까지 정리되었습니다.

## 5. 아직 남아 있는 실작업 성격의 부분

이제부터는 “화면 설계”보다 **서비스 연결** 단계에 가깝습니다.

대표적으로는 아래입니다.

### 작업 내역

* 실제 검색 조건 반영
* 정렬 반영
* 체크박스 선택 상태 처리
* 재작업/삭제 서비스 연결
* 연령대별 Step1 결과 판단과 재작업 진입 연결

### 환경설정

* 저장/불러오기 실제 연결 확인
* 메뉴 설정 저장값을 실제 사이드 메뉴 렌더링에 반영
* 기본 이메일/PIN/SMTP/AI 모드 연동 안정화
* 더미 처리된 일부 기능 연결 여부 검토

  * 로고 탐색기
  * 로고 미리 보기 등

즉 지금부터는 “UI 설계 검토”보다 “UI와 서비스 연결 점검” 단계입니다.

## 6. 지금 기준으로 가장 중요한 확인 포인트

서비스 연결 전에 아래를 확인하면 좋습니다.

### 작업 내역

* 재작업은 정확히 어디까지 복원할 것인가
* 선택 1건 기준 재작업 흐름이 맞는가
* 삭제는 일괄 삭제가 맞는가

### 환경설정

* 메뉴 설정 저장값을 앱 시작 시점에 읽을지
* 메뉴 숨김 시 실제 접근 로직도 막을지, 단순 숨김만 할지
* 표시명 변경이 사이드 메뉴 외 다른 화면 제목에도 반영될지

이건 구현 전에 한 번만 더 정리하면 됩니다.

## 7. 한 줄 결론

지금까지의 작업을 정리하면 이렇게 볼 수 있습니다.

* **작업 내역**은 조회형 상세 화면이 아니라 **선택형 재작업 화면**으로 정리됨
* **환경설정**은 장년 QT Step 계열 구조에 맞춰 거의 정리됨
* **메뉴 설정**은 별도 탭 + 표 방식으로 가는 것이 맞다고 정리됨
* **CSS**도 새 구조 중심으로 재정리되는 단계까지 왔음
* 이제 다음 단계는 **화면 설계 확정 후 서비스 연결**입니다

**“서비스 연결 전에 확인해야 할 체크리스트”** 형태로 아주 짧게 정리해 드리겠습니다.
