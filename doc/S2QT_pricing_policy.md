# S2QT 무료·유료·구독 정책 정리

## 1. 개요
S2QT는 설교 콘텐츠를 입력받아 QT(Quiet Time) 형식으로 전환하고, 이를 PDF 또는 교재 형태로 출력할 수 있는 서비스이다.

본 문서는 S2QT의 수익모델을 다음 3가지 축으로 정리한다.

- 무료 정책: 제한된 범위 내에서 핵심 경험 제공
- 유료 정책: 프리미엄 템플릿 기반 출력물 과금
- 구독 정책: 플랫폼이 제공하는 LLM을 활용하는 SaaS형 과금

핵심 방향은 다음과 같다.

- 무료는 “설교 → QT 전환 경험”을 제공한다.
- 유료는 “더 좋은 출력물과 목적형 템플릿”에 과금한다.
- 구독은 “플랫폼 제공 LLM을 통한 편리한 생성 경험”에 과금한다.

---

## 2. 정책 설계 원칙

### 2.1 무료 정책의 목적
무료 정책은 사용자가 S2QT의 핵심 가치를 충분히 체험할 수 있도록 하는 데 목적이 있다.

즉, 사용자가 실제로 다음을 경험해야 한다.

- 설교가 QT로 전환되는 경험
- QT가 PDF 형태로 출력되는 경험
- 서비스의 기본 품질 확인

### 2.2 유료 정책의 목적
유료 정책은 단순 기능 잠금 해제가 아니라, 실제 활용 가치가 높은 결과물에 과금하는 방향으로 설계한다.

즉, 유료의 핵심은 다음과 같다.

- 더 완성도 높은 템플릿 제공
- 목적별 교재/묵상집 출력 지원
- 실제 배포·활용 가능한 PDF 결과물 제공

### 2.3 구독 정책의 목적
구독 정책은 사용자가 직접 LLM API를 설정하지 않고, 플랫폼이 제공하는 LLM을 월 사용량 범위 내에서 바로 활용할 수 있도록 하는 것이다.

이는 다음과 같은 사용자에게 적합하다.

- API Key 사용이 어려운 일반 사용자
- 별도 설정 없이 바로 사용하고 싶은 사용자
- 반복적으로 설교-QT 생성을 이용하는 사용자

---

## 3. 무료 정책

### 3.1 기본 방향
무료 정책은 “체험 가능한 수준”으로 제공하되, 서비스의 핵심 가치만 확인할 수 있도록 제한한다.

### 3.2 무료 제공 범위
- 월 4~5회 사용 가능
- 주별 1회 수준으로 사용 가능
- 1일 QT만 생성 가능
- 베이직 템플릿으로 PDF 출력 가능

### 3.3 무료 정책의 의미
무료 사용자는 S2QT의 핵심 흐름을 경험할 수 있다.

- 설교 입력
- 1일 QT 생성
- 베이직 템플릿 적용
- PDF 출력

즉, 무료 정책은 “사용 경험 제공”에 초점을 둔다.

---

## 4. 유료 정책

### 4.1 기본 방향
유료 정책은 구독보다는 “프리미엄 결과물”에 대한 과금 모델로 설계한다.

즉, 사용자는 기본 생성 자체보다도, 더 높은 품질의 출력물과 목적별 템플릿에 비용을 지불하게 된다.

### 4.2 유료 제공 범위
- 유료 템플릿 구매 후 PDF 출력 가능
- 프리미엄 템플릿 기반 QT 출력 가능
- 목적별 템플릿(예: 개인 묵상용, 소그룹용, 교재용 등) 적용 가능

### 4.3 유료 정책의 의미
유료 정책은 단순히 “더 예쁜 디자인”이 아니라, 실제 사용 목적에 맞는 결과물 제공에 초점을 둔다.

즉, 유료는 다음과 같은 가치를 제공한다.

- 더 완성도 높은 레이아웃
- 배포·인쇄에 적합한 구성
- 교재/묵상집 형태의 출력물
- 사용 목적에 맞는 템플릿 선택

### 4.4 유료의 핵심 메시지
유료 정책은 “기능 해제”가 아니라 “활용 가능한 결과물”에 대한 과금이다.

---

## 5. 구독 정책

### 5.1 기본 방향
구독 정책은 사용자가 직접 LLM API를 설정하는 방식이 아니라, S2QT가 제공하는 LLM을 이용하는 서비스형(SaaS) 모델이다.

즉, 사용자는 별도의 API Key 없이도 바로 설교-QT 생성 기능을 사용할 수 있다.

### 5.2 구독 정책의 의미
구독형은 다음과 같은 사용자 경험을 제공한다.

- 회원가입 후 바로 사용 가능
- API 설정 불필요
- 플랫폼이 제공하는 LLM 사용
- 월 사용량 범위 내 QT 생성 가능

### 5.3 구독형의 필요성
구독형은 사용 편의성 때문이기도 하지만, 데이터 재활용 기반 운영 구조를 통해 효율성을 확보할 수 있다는 점에서 중요하다.

S2QT는 매 요청마다 새로 생성하는 방식이 아니라, 다음 로직을 우선 적용한다.

- 기존에 생성된 설교/QT/성구/전사 결과가 있는지 조회
- 있으면 기존 데이터를 재사용
- 없을 경우에만 LLM으로 신규 생성

즉, 구독형의 운영 원칙은 다음과 같다.

> 있는 데이터는 재사용하고, 없는 데이터만 생성한다.

이를 통해 다음 효과를 기대할 수 있다.

- LLM 사용량 절감
- 응답 속도 향상
- 품질 안정화
- 콘텐츠 자산 축적

### 5.4 구독형의 핵심 구조
구독형 S2QT는 단순 LLM 호출 서비스가 아니라, 설교-QT 콘텐츠를 자산화하고 재활용하는 플랫폼 구조를 지향한다.

처리 흐름은 다음과 같다.

1. 사용자가 설교 입력
2. 동일 또는 유사 설교 데이터 존재 여부 확인
3. 기존 전사/QT/성구 데이터가 있으면 재사용
4. 필요한 경우에만 신규 생성
5. 결과를 다시 저장하여 이후 재사용 가능하도록 함

### 5.5 구독형의 핵심 메시지
구독형은 “AI를 직접 설정하는 도구”가 아니라, “플랫폼이 제공하는 생성 서비스를 바로 사용하는 방식”이다.

---

## 6. 정책 간 관계
S2QT의 무료·유료·구독 정책은 서로 대체 관계가 아니라, 각기 다른 가치를 제공하는 구조로 설계한다.

### 무료
- 핵심 경험 제공
- 제한된 사용 횟수
- 1일 QT
- 베이직 PDF 출력

### 유료
- 프리미엄 템플릿 기반 출력
- 목적형 PDF/교재 결과물
- 출력물 품질에 대한 과금

### 구독
- 플랫폼 제공 LLM 사용
- API 설정 없는 편리한 생성 경험
- 월 사용량 기반 서비스 이용

즉, 각 정책의 중심 가치는 다음과 같이 구분된다.

- 무료: 체험
- 유료: 출력물
- 구독: 생성 편의성

---

## 7. 문서용 요약
S2QT는 무료 사용자에게 월 4~5회의 1일 QT 생성과 베이직 템플릿 기반 PDF 출력을 제공하고, 유료 사용자에게는 프리미엄 템플릿 구매를 통한 고급 PDF·교재 출력 기능을 제공한다. 또한 구독형 서비스는 사용자가 별도의 LLM API를 직접 연결하지 않고, 플랫폼이 제공하는 LLM을 월 사용량 범위 내에서 이용할 수 있는 SaaS 방식으로 구성한다. 이때 구독형은 기존 설교-QT 데이터를 우선 재활용하고 필요한 경우에만 신규 생성하는 구조를 통해 LLM 사용량을 효율적으로 관리한다.

---

## 8. 한 줄 정리
S2QT의 수익모델은 무료 체험, 프리미엄 템플릿 기반 유료 출력, 플랫폼 제공 LLM 기반 구독형 서비스를 결합한 구조이며, 구독형은 데이터 재활용 중심의 운영 방식으로 효율성을 확보한다.
