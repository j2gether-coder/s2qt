이제는 바로 구현 가능한 수준으로 **실제 컴포넌트 목록**을 정리하겠습니다.

기준은 지금까지 합의한 구조입니다.

* 좌측 메뉴는 단순 네비게이션
* 우측은 선택한 화면의 작업 영역
* 전사는 공통 1회
* audience별 화면은 Step1~4 반복
* 내부 데이터는 JSON 중심

# QT 작성기 실제 컴포넌트 목록 초안

## 1. 최상위 레이아웃 컴포넌트

## 1.1 `AppShell`

전체 화면의 최상위 레이아웃 컨테이너

### 역할

* 좌측 메뉴와 우측 작업 영역 배치
* 현재 선택된 메뉴 상태 관리
* 공통 헤더 표시

### 하위 컴포넌트

* `AppHeader`
* `SideNav`
* `MainWorkspace`

---

## 1.2 `AppHeader`

상단 헤더 영역

### 표시 항목

* 앱명: QT 작성기
* 현재 설교 제목
* 공통 저장 상태
* 마지막 저장 시각

### 버튼

* 저장
* 새로고침

### 상태값

* `currentTitle`
* `sourceSaved`
* `lastSavedAt`

---

## 1.3 `SideNav`

좌측 메뉴 영역

* 전사
* 장년

# 11. 정리

이 컴포넌트 목록의 핵심은 다음입니다.

* **좌측은 단순 메뉴**
* **전사 화면은 공통 원문 준비**
* **audience 화면은 Step1~4 재사용 구조**
* **Step3는 JSON 구조화 편집**
* **Step4는 JSON 기반 출력**

이제 다음 단계는 자연스럽게 두 가지 중 하나입니다.

하나는 **컴포넌트별 props/state/event 표**로 내려가는 것이고,
다른 하나는 **실제 화면 구현 순서**를 정하는 것입니다.

프로젝트 진행용으로는 다음에 **구현 우선순위와 개발 순서표**를 정하는 게 가장 좋습니다.
