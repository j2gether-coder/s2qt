# 2. 보안 원칙 확정

## 2-1. PIN 정책

* 앱 시작 시 로그인 없음
* Step1 자동 LLM 사용 전 PIN 확인
* 환경설정에서 민감정보 수정 전 PIN 확인
* PIN은 해시 저장
* PIN은 평문 재조회 불가

## 2-2. 세션 정책

* 한 번 PIN 인증 성공 시 현재 앱 실행 세션 동안 unlock 유지
* 같은 세션에서는 연령별 Step1에서 재인증 없음
* 앱 종료 시 잠금 해제

## 2-3. 가상키보드 정책

* 숫자 전용 PIN 입력
* **숫자 0~9 버튼은 모달이 열릴 때마다 랜덤 배치**
* 입력값은 마스킹
* `지우기 / 전체삭제 / 확인 / 취소` 버튼은 고정 위치 권장

## 2-4. 민감정보 정책

* `llm.api_key` 암호화 저장
* `smtp.password` 암호화 저장
* UI에서는 평문 표시 안 함
* 화면에는 `등록됨`, `********` 정도로만 표시

# 3. 로컬 DB 확정 테이블

## 3-1. `app_settings`

```sql
CREATE TABLE IF NOT EXISTS app_settings (
    setting_key   TEXT PRIMARY KEY,
    setting_value TEXT,
    value_type    TEXT NOT NULL DEFAULT 'text',
    is_secret     INTEGER NOT NULL DEFAULT 0,
    setting_group TEXT NOT NULL DEFAULT 'general',
    updated_at    TEXT NOT NULL
);
```

## 3-2. `history_master`

```sql
CREATE TABLE IF NOT EXISTS history_master (
    id           INTEGER PRIMARY KEY AUTOINCREMENT,
    title        TEXT NOT NULL,
    bible_text   TEXT NOT NULL,
    hymn         TEXT,
    preacher     TEXT,
    church_name  TEXT,
    sermon_date  TEXT,
    created_at   TEXT NOT NULL,
    updated_at   TEXT NOT NULL
);
```

## 3-3. `history_step1`

```sql
CREATE TABLE IF NOT EXISTS history_step1 (
    id                INTEGER PRIMARY KEY AUTOINCREMENT,
    history_id        INTEGER NOT NULL,
    audience          TEXT NOT NULL,
    step1_result_json TEXT NOT NULL,
    created_at        TEXT NOT NULL,
    updated_at        TEXT NOT NULL,
    FOREIGN KEY (history_id) REFERENCES history_master(id)
);
```

# 7. 실무적으로 확정된 한 줄 요약

**S2QT 로컬형은 SQLite + Go 서비스 기반으로 가고, 보안은 PIN 해시 + 민감값 암호화 + 랜덤 숫자 가상키보드로 처리하며, DB 이미지 은닉 방식은 채택하지 않는다.**

다음으로는 이 확정안 기준으로 **Go 코드 골격 4개 파일부터 작성**하는 것이 가장 자연스럽습니다.
