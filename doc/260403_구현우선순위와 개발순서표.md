좋습니다.
이제는 실제 착수 기준이 되는 **구현 우선순위와 개발 순서표**를 정하면 됩니다.

핵심 원칙은 간단합니다.

* 먼저 **전체 구조를 움직이게 만들고**
* 그다음 **JSON 중심 흐름을 고정하고**
* 마지막에 **문서 품질과 확장 기능**을 올립니다

즉 처음부터 DOCX/PPTX 품질까지 다 잡으려 하지 말고,
**전사 → JSON 생성 → 편집 → HTML/PDF 출력**의 기본 축을 먼저 완성해야 합니다.

# QT 작성기 구현 우선순위 및 개발 순서표

## 1. 구현 우선순위 원칙

### 1차 목표

**공통 원문 1회 준비 + audience별 QT JSON 생성/편집 + HTML/PDF 출력**
여기까지를 하나의 완성 단위로 봅니다.

### 2차 목표

**DOCX/PPTX 연결 및 품질 개선**

### 3차 목표

**템플릿 확장, audience 비교, 복제 생성, 운영 편의 기능**

---

## 2. 전체 개발 단계

전체 개발은 아래 6단계로 나누는 것이 좋습니다.

### Phase 1. 공통 원문 준비 기반 구축

### Phase 2. audience별 QT JSON 생성 구조 구축

### Phase 3. JSON 기반 편집 화면 구축

### Phase 4. HTML/PDF 산출물 구축

### Phase 5. DOCX/PPTX 확장

### Phase 6. 운영 편의 및 고도화

---

# 3. Phase 1. 공통 원문 준비 기반 구축

## 목표

전사/텍스트 입력/공통 메타 입력을 통해 **공통 원문 모델**을 저장할 수 있어야 합니다.

## 구현 항목

### 1-1. 좌측 메뉴 기본 레이아웃

구현 대상:

* `AppShell`
* `AppHeader`
* `SideNav`
* `MainWorkspace`

완료 기준:

* 좌측에 전사/장년/청년/중고등부/어린이 메뉴 노출
* 메뉴 선택 시 우측 화면이 전환됨

---

### 1-2. 전사 화면 기본 UI

구현 대상:

* `TranscriptionWorkspace`
* `SourceTypeSelector`
* `SourceInputPanel`
* `CommonMetadataForm`
* `TranscriptViewer`
* `TranscriptionActionBar`

완료 기준:

* 동영상/오디오/텍스트 입력 원천 선택 가능
* 제목/본문 성구 등 메타 입력 가능
* raw/cleaned text 표시 가능

---

### 1-3. 공통 원문 모델 저장

구현 대상:

* `sermon_source.json` 저장/불러오기
* source_id 생성
* source_type 저장
* metadata 저장
* transcript 저장

완료 기준:

* 저장 버튼 클릭 시 공통 원문 JSON 파일 생성
* 다시 열었을 때 동일 값 복원 가능

---

### 1-4. 전사 기능 연결

구현 대상:

* 동영상 → 다운로드/오디오 추출/전사
* 오디오 → 전사
* 텍스트 → raw_text 반영
* cleaned_text 생성

완료 기준:

* 입력 원천별로 공통 원문 확보 가능
* 텍스트 입력은 전사 없이도 저장 가능

---

## Phase 1 산출물

* 좌측 메뉴 레이아웃
* 전사 화면
* 공통 원문 JSON 저장 구조

---

# 4. Phase 2. audience별 QT JSON 생성 구조 구축

## 목표

공통 원문을 기반으로 audience별 QT output JSON을 생성할 수 있어야 합니다.

## 구현 항목

### 2-1. audience 화면 공통 프레임

구현 대상:

* `AudienceWorkspace`
* `AudienceHeader`
* `AudienceStepTabs`
* `AudienceStepContent`

완료 기준:

* 장년/청년/중고등부/어린이 메뉴 진입 시 공통 Step 구조 표시
* 각 audience 화면이 독립 동작

---

### 2-2. Step1 생성 조건 확인 화면

구현 대상:

* `GenerationOptionsStep`
* `AudienceOptionsForm`
* `SourceSummaryCard`
* `GenerationOptionsActionBar`

완료 기준:

* audience, template_id, 문체 옵션 등 저장 가능
* 공통 원문 요약 표시 가능

---

### 2-3. 프롬프트 관리 구조

구현 대상:

* `qt_prompt_base`
* `qt_prompt_adult`
* `qt_prompt_young_adult`
* `qt_prompt_teen`
* `qt_prompt_child`

완료 기준:

* audience별로 프롬프트 조합 가능
* base + audience 규칙 + request JSON 결합 가능

---

### 2-4. Step2 AI QT 생성

구현 대상:

* `AiGenerationStep`
* `AiGenerationActionBar`
* `AiGenerationStatusPanel`
* output JSON 검증

완료 기준:

* AI 호출 후 audience별 QT output JSON 생성
* `qt_adult.json`, `qt_young_adult.json` 등 저장 가능

---

### 2-5. QT output JSON 검증기

구현 대상:

* sections 4개 검증
* message blocks 6개 검증
* reflection list 3개 검증
* prayer paragraph 1개 검증

완료 기준:

* 잘못된 구조는 저장 차단 또는 경고 표시

---

## Phase 2 산출물

* audience별 Step1/2
* 프롬프트 조합 구조
* QT output JSON 생성 및 저장

# 5. Phase 3. JSON 기반 편집 화면 구축

## 목표

생성된 QT JSON을 구조화된 화면으로 수정할 수 있어야 합니다.

## 구현 항목

### 3-1. Step3 전체 구조

구현 대상:

* `QtEditorStep`
* `StructuredEditorPane`
* `HtmlPreviewPane`
* `EditorActionBar`

완료 기준:

* 좌측 편집 / 우측 미리보기 구조 구현

---

### 3-2. 구조화 편집 컴포넌트

구현 대상:

* `QtBasicInfoEditor`
* `QtSummaryEditor`
* `QtMessageEditor`

### 3-3. 저장/복원 기능

구현 대상:

* 편집 저장

완료 기준:

* 수정 후 저장 가능
* 재진입 시 마지막 저장본 복원 가능

# 6. Phase 4. HTML/PDF 산출물 구축

## 목표

* 기존 HTML/CSS 자산 재사용

완료 기준:

* summary/message/reflection/prayer가 HTML로 일관되게 출력됨

---

### 4-2. HTML Preview 연동

구현 대상:

* Step3 우측 preview

* 편집 내용이 preview에 반영됨

---

구현 대상:

* `OutputGenerationStep`

### 4-4. Step4 PDF 생성

# 7. Phase 5. DOCX/PPTX 확장

## 목표

Python worker를 이용해 DOCX/PPTX를 생성합니다.

## 구현 항목

### 5-1. Python worker 입출력 규격 확정

구현 대상:

* input: QT output JSON
* output: docx/pptx 파일 경로

완료 기준:

* Go/Wails와 Python 간 파일 기반 연동 가능

---

### 5-2. DOCX POC

구현 대상:

* `word_python.py`
* JSON → DOCX 생성
* 섹션 스타일 반영

완료 기준:

* Word에서 자연스럽게 열리고 섹션 구분이 명확함

---

### 5-3. PPTX POC

구현 대상:

* `ppt_python.py`
* JSON → PPTX 생성
* 가능하면 템플릿 기반

완료 기준:

* 복구 경고 없이 열리거나, 최소한 안정적 생성 구조 확보

---

### 5-4. Step4 DOCX/PPTX 연결

구현 대상:

* Go에서 Python worker 호출
* 파일 생성 상태 표시
* 열기 버튼 연결

완료 기준:

* Step4에서 DOCX/PPTX 생성 가능

---

## Phase 5 산출물

* DOCX 생성
* PPTX 생성
* Step4 통합 출력 기능
