## 1. 적용 범위

본 확정본은 **로컬형 S2QT** 기준이다.

포함 범위:
- 환경설정(Settings)
- PIN 보안
- History
- Step1 결과 재사용

제외 범위:
- 서버형 로그인/구독/결제
- 서버 동기화
- Step2 결과 저장
- Step3 결과 저장
- 전사문(raw_text) 저장

---

## 2. 로컬형 저장 원칙

### 2.1 환경설정
SQLite `app_settings` 테이블에 저장한다.

저장 대상:
- AI 설정
- SMTP 설정
- 교회/브랜드 설정
- 안내/라이선스 표시용 설정
- PIN 관련 설정

### 2.2 History
저장 대상:
- 기본정보
- Step1 결과 JSON

저장 제외:
- 전사문
- Step2 결과
- Step3 결과
- 산출물 경로

### 2.3 보안
- 앱 실행 시 로그인 없음
- 민감 기능 사용 시 PIN 확인
- PIN은 해시 저장
- API KEY / SMTP 비밀번호는 암호화 저장
- 세션 unlock은 프론트 메모리 상태에서 관리

# 3. SQLite DDL

## 3.1 app_settings

```sql
CREATE TABLE IF NOT EXISTS app_settings (
    setting_key   TEXT PRIMARY KEY,
    setting_value TEXT,
    value_type    TEXT NOT NULL DEFAULT 'text',
    is_secret     INTEGER NOT NULL DEFAULT 0,
    setting_group TEXT NOT NULL DEFAULT 'general',
    updated_at    TEXT NOT NULL
);
````

### 설명

* `setting_key`: 설정 키
* `setting_value`: 설정 값
* `value_type`: text, password, number, boolean, url, multiline
* `is_secret`: 민감값 여부
* `setting_group`: 설정 그룹
* `updated_at`: 수정 시각

---

## 3.2 history_master

```sql id="3i0a3e"
CREATE TABLE IF NOT EXISTS history_master (
    id           INTEGER PRIMARY KEY AUTOINCREMENT,
    title        TEXT NOT NULL,
    bible_text   TEXT NOT NULL,
    hymn         TEXT,
    preacher     TEXT,
    church_name  TEXT,
    sermon_date  TEXT,
    created_at   TEXT NOT NULL,
    updated_at   TEXT NOT NULL
);
```

### 설명

* 설교 1건의 기본정보 저장
* 로컬 재작업의 기준 엔터티

## 3.3 history_step1

```sql id="jp17pa"
CREATE TABLE IF NOT EXISTS history_step1 (
    id                INTEGER PRIMARY KEY AUTOINCREMENT,
    history_id        INTEGER NOT NULL,
    audience          TEXT NOT NULL,
    step1_result_json TEXT NOT NULL,
    created_at        TEXT NOT NULL,
    updated_at        TEXT NOT NULL,
    FOREIGN KEY (history_id) REFERENCES history_master(id)
);
```

### 설명

* 연령대별 Step1 결과 JSON 저장
* `audience`: adult / young_adult / teen / child

# 7. 로컬형 데이터 흐름 기준

## 7.1 환경설정 저장

1. 프론트 탭 화면에서 설정 입력
2. 일반값은 `SaveAppSettings()`
3. 민감값은 `SaveSecretSetting()`
4. 저장 후 toast 표시

## 7.2 PIN 검증

1. Step1 자동 LLM 실행 전 또는 민감정보 수정 전
2. PIN 모달 표시
3. `VerifyPin(pin)` 호출
4. 성공 시 세션 unlock
5. 같은 앱 실행 세션에서는 재인증 없음

## 7.3 History 저장

1. Step1 결과 JSON 저장 성공
2. 기본정보 + audience + step1 JSON으로 `SaveHistory()` 호출
3. 목록 화면에서 조회 가능
4. 선택 시 `GetHistory() + GetHistoryStep1()`으로 불러오기
5. Step2부터 다시 작업
