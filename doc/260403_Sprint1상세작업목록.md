# Sprint 1 목표

**앱의 기본 골격을 만들고, 공통 원문 준비 화면에서 입력 원천(video/audio/text), 메타데이터, 원문 텍스트를 저장할 수 있도록 한다.**

즉 Sprint 1이 끝나면 최소한 아래는 되어야 합니다.

* 앱 기본 레이아웃 표시
* 좌측 메뉴 동작
* 전사 화면 표시
* 입력 원천 선택 가능
* 공통 메타 입력 가능
* raw/cleaned text 입력 및 표시 가능
* `sermon_source.json` 저장 가능
* 저장된 데이터 다시 불러오기 가능

아직 이 단계에서는

* audience별 Step 화면
* LLM 호출
* HTML preview
* DOCX/PPTX
  는 하지 않습니다.

# Sprint 1 상세 작업목록

## 1. 작업 범위

Sprint 1 범위는 아래 4개입니다.

1. 앱 기본 레이아웃
2. 좌측 메뉴
3. 전사 화면
4. 공통 원문 JSON 저장/불러오기

---

## 2. 완료 기준

Sprint 1 완료 기준은 아래와 같습니다.

* 앱 실행 시 좌측 메뉴와 우측 작업 영역이 보인다
* 좌측에서 `전사` 메뉴 선택 시 공통 원문 준비 화면이 보인다
* 입력 원천으로 `동영상 / 오디오 / 텍스트` 중 하나를 선택할 수 있다
* 제목/본문 성구/찬송/설교자/교회명/설교일 입력이 가능하다
* raw_text / cleaned_text를 화면에서 입력 또는 확인할 수 있다
* 저장 시 `sermon_source.json` 파일이 생성된다
* 다시 실행해도 저장값을 불러올 수 있다

## Task 9. 공통 원문 JSON 스키마 정의

### 목적

저장할 데이터 구조를 코드로 명확히 한다.

### 저장 스키마

```json
{
  "source_id": "",
  "source_type": "",
  "source_ref": {
    "url": "",
    "file_path": ""
  },
  "metadata": {
    "title": "",
    "bible_text": "",
    "hymn": "",
    "preacher": "",
    "church_name": "",
    "sermon_date": ""
  },
  "transcript": {
    "raw_text": "",
    "cleaned_text": ""
  }
}
```

### 구현 항목

* Go struct 정의
* 프론트 상태와 매핑 규칙 정의

### 완료 기준

# 7. Sprint 1 종료 시점 데모 시나리오

10. 저장값 자동 복원 확인

이 시나리오가 되면 Sprint 1은 성공입니다.
