
# S2QT 작업내역 / 환경설정 서비스 연동 체크 및 연동 리스트

## 1. 목적

현재 완성된 화면:

* 작업 내역
* 환경설정

  * 기본 정보
  * 부가 기능
  * 메뉴 설정
  * 안내

에 대해,
이미 작성된 Go 파일과 실제로 어떤 항목을 연결해야 하는지 점검하고,
빠짐없이 연결 작업을 진행하기 위한 목록입니다.

---

# 2. 공통 점검 원칙

## 2-1. 먼저 확인할 것

각 화면에서 다음 3가지를 구분해서 봅니다.

* 이미 Go 함수가 있음
* Go 함수는 있으나 화면과 매핑이 안 됨
* Go 함수가 아직 없음

## 2-2. 연동 기준

연동은 아래 순서로 진행하는 것이 좋습니다.

1. 조회(load)
2. 저장(save)
3. 상태 반영(render)
4. 예외 처리(error)
5. 사용자 피드백(toast / inline message)

# 3. 환경설정 연동 체크 리스트

---

## 3-1. 기본 정보 탭

### 화면 항목

* 기본 이메일
* PIN 설정
* 교회/브랜드 설정

  * 교회명
  * 홈페이지 URL
  * 로고 경로
  * footer 문구

### 현재 연동 대상 Go 함수

* `LoadAppSettingsByGroup`
* `SaveAppSettings`
* `IsPinEnabled`
* `GetPinLength`
* `SetupPin`
* `ChangePin`

### 체크 항목

* [ ] `user` 그룹 설정이 정상 조회되는가
* [ ] `church` 그룹 설정이 정상 조회되는가
* [ ] 이메일 저장 시 `user.email`에 정상 저장되는가
* [ ] PIN 등록 시 이메일 선검증 후 처리되는가
* [ ] PIN 변경 시 현재 PIN 검증이 정상 동작하는가
* [ ] PIN 길이(4/6) 정책이 UI와 일치하는가
* [ ] 교회명, URL, 로고 경로, footer 문구가 `church.*`로 정상 저장되는가
* [ ] 저장 후 재조회 시 값이 유지되는가

### 추가 확인

* [ ] `church.logo_path`는 현재 문자열 저장만 할지
* [ ] 나중에 파일 탐색기 연결 시 별도 Go 함수가 필요한지
* [ ] 로고 미리보기는 파일 경로 기반으로 프론트 처리 가능한지

---

## 3-2. 부가 기능 탭

### 화면 항목

* AI 사용 방식

  * 수동
  * 원격
  * 로컬
* 원격 AI API KEY
* SMTP

  * 사용 여부
  * 발신 이메일 표시
  * SMTP 서버
  * 포트
  * 보안 방식
  * 사용자명
  * 앱 비밀번호

### 현재 연동 대상 Go 함수

* `LoadAppSettingsByGroup`
* `SaveAppSettings`
* `HasSecretValue`
* `SaveSecretSettingWithPin`

### 체크 항목

#### AI

* [ ] `llm.mode` 또는 `ai.mode`를 정상 조회하는가
* [ ] 기본값이 `manual`로 잘 들어가는가
* [ ] 라디오 선택 후 저장 시 `llm.mode`, `ai.mode`가 정상 저장되는가
* [ ] `remote` 선택 시 API KEY 등록 상태가 정상 표시되는가
* [ ] API KEY 수정/저장 전에 PIN 검증이 호출되는가
* [ ] `llm.api_key` 비밀값 저장이 정상 동작하는가

#### SMTP

* [ ] `smtp.enabled` 조회/저장이 정상인가
* [ ] `smtp.host`, `smtp.port`, `smtp.security`, `smtp.username` 조회/저장이 정상인가
* [ ] `smtp.password` 저장 전에 PIN 검증이 정상 호출되는가
* [ ] `smtp.password`가 비밀값으로 정상 저장되는가
* [ ] 발신 이메일은 `user.email`을 기준으로 정상 표시되는가
* [ ] 기본 이메일이 없을 때 안내 문구가 정상 표시되는가

### 추가 확인

* [ ] SMTP 테스트 발송 기능이 필요한지
* [ ] 필요 시 별도 Go 함수 추가 여부

  * 예: `SendTestMail(...)`

## 3-3. 메뉴 설정 탭

### 화면 항목

* 사용 여부
* 기본 메뉴
* 표시명

대상:

* adult
* young_adult
* teen
* child

### 현재 연동 대상 Go 함수

* `LoadAppSettingsByGroup`
* `SaveAppSettings`

### 저장 키

* `menu.adult.visible`
* `menu.adult.label`
* `menu.young_adult.visible`
* `menu.young_adult.label`
* `menu.teen.visible`
* `menu.teen.label`
* `menu.child.visible`
* `menu.child.label`

### 체크 항목

* [ ] `menu` 그룹 설정 조회가 정상인가
* [ ] 표시 여부 체크값이 boolean 문자열로 정상 저장되는가
* [ ] 표시명이 비어 있으면 기본값으로 대체되는가
* [ ] 최소 1개 메뉴 표시 조건이 정상 검증되는가
* [ ] 저장 후 다시 열었을 때 값이 유지되는가

### 매우 중요한 연동 항목

* [ ] 사이드 메뉴 렌더링 로직이 `menu.*.visible`, `menu.*.label`을 읽도록 연결되었는가
* [ ] 숨김 처리된 메뉴는 실제로 메뉴에서 보이지 않는가
* [ ] 표시명 변경 시 사이드 메뉴 텍스트가 바뀌는가
* [ ] 내부 audience 키는 그대로 유지되는가

### 추가 확인

* [ ] 메뉴 숨김 시 해당 메뉴 진입 자체를 막을지
* [ ] 아니면 단순히 사이드 메뉴에서만 숨길지 정책 확인

## 3-4. 안내 탭

### 화면 항목

* 라이선스
* 도움말
* 사용 가이드

### 현재 상태

* 기본 HTML 내용 하드코딩

### 체크 항목

* [ ] 현재는 프론트 기본 문구로 충분한가
* [ ] 추후 Go에서 md/html 파일을 읽어오게 할 것인가

### 필요 시 추가 Go 함수

예:

* `LoadGuideDocument(sectionId string)`

현재는 필수 아님.

# 4. 작업 내역 연동 체크 리스트

---

## 4-1. 작업 내역 조회

### 화면 항목

* 검색 조건

  * 검색어
  * 연령대
  * 정렬 기준
  * 정렬 방향
* 작업 버튼

  * 재작업
  * 삭제
* 테이블

  * 체크박스
  * 제목
  * 본문 성구
  * 저장일
  * 연령대

### 현재 연동 대상 Go 함수

* `ListHistory`
* `GetHistory`
* `GetHistoryStep1`
* `DeleteHistory`

### 체크 항목

* [ ] `ListHistory()`가 목록 기본정보를 충분히 주는가
* [ ] `title`, `bibleText`, `createdAt` 값이 목록에 포함되는가
* [ ] 저장일이 `createdAt` 기준으로 정상 표시되는가
* [ ] 연령대별 Step1 존재 여부를 `GetHistoryStep1`로 판단 가능한가
* [ ] 화면 로딩 시 목록+연령대 가능 여부 조회가 정상 수행되는가

---

## 4-2. 검색 / 정렬

### 현재 구조

프론트에서 필터/정렬 수행 중

### 체크 항목

* [ ] 검색어가 제목/본문 성구 기준으로 정상 필터링되는가
* [ ] 연령대 필터가 Step1 존재 여부 기준으로 정상 작동하는가
* [ ] 정렬 기준

  * 저장일
  * 제목
  * 본문 성구
    가 정상 동작하는가
* [ ] 오름차순 / 내림차순이 정상 동작하는가

### 추가 검토

* 데이터가 많아지면 서버측 검색/정렬로 옮길 필요가 있는가
* 필요 시 Go 함수 추가

  * 예: `SearchHistory(filter...)`

현재는 프론트 처리로 시작해도 괜찮습니다.

---

## 4-3. 재작업

### 현재 의도

* 선택 1건만 가능
* 해당 작업의 기본정보 + Step1 결과를 복원
* Step2부터 다시 진행

### 현재 연동 대상 Go 함수

* `GetHistory`
* `GetHistoryStep1`

### 체크 항목

* [ ] 선택 1건만 허용되는가
* [ ] 선택된 작업의 기본정보를 `appState.source.basicInfo`에 복원하는가
* [ ] Step1 JSON을 `appState.historySelected`에 저장하는가
* [ ] 해당 audience 메뉴로 이동하는가
* [ ] `setAudienceStep(audienceId, "step2")`가 정상 동작하는가
* [ ] 화면 재마운트 후 Step2가 실제로 복원 데이터를 읽는가

### 가장 중요한 정책 확인

* [ ] 재작업 시 audience를 어떻게 결정할 것인가

  * 첫 번째 가능한 audience 자동 선택
  * 또는 사용자에게 선택하게 할 것인가

현재 자동 선택이면 구현은 단순하지만,
나중에는 선택 팝업이 필요할 수 있습니다.

---

## 4-4. 삭제

### 현재 연동 대상 Go 함수

* `DeleteHistory`

### 체크 항목

* [ ] 체크 1건 이상 선택 시 삭제 가능한가
* [ ] 다건 삭제가 순차적으로 정상 수행되는가
* [ ] 삭제 후 목록 재조회가 정상 되는가
* [ ] 선택 상태가 초기화되는가
* [ ] 삭제 후 토스트가 정상 표시되는가

### 추가 검토

* [ ] 다건 삭제용 Go 함수가 필요할지

  * 예: `DeleteHistories(ids []int)`
    현재는 반복 호출로도 가능

# 5. 사이드 메뉴 실제 반영 체크 리스트

이 부분은 메뉴 설정 탭과 연결되는 핵심입니다.

### 대상

앱의 좌측 메뉴 렌더링 로직

### 체크 항목

* [ ] 앱 시작 시 `menu` 그룹 설정을 읽는가
* [ ] 각 audience 메뉴에 대해

  * visible 적용
  * label 적용
    이 되는가
* [ ] 표시명 변경 시 메뉴 텍스트가 실제 바뀌는가
* [ ] 숨김 메뉴는 사이드 메뉴에서 사라지는가
* [ ] 내부 선택 상태와 audience ID는 기존 그대로 유지되는가

### 필요 시 추가 작업

* 사이드 메뉴 구성용 helper 함수 분리
* 예: `buildAudienceMenuConfig(settings)`

# 6. 서비스 연결 전 확인해야 할 Go 파일 목록

아래 파일들을 먼저 확인하면 좋습니다.

## 환경설정 관련

* app settings 저장/조회 Go 파일
* secret setting 저장 Go 파일
* PIN 관련 Go 파일

확인 함수:

* `LoadAppSettingsByGroup`
* `SaveAppSettings`
* `HasSecretValue`
* `SaveSecretSettingWithPin`
* `IsPinEnabled`
* `GetPinLength`
* `SetupPin`
* `ChangePin`

## 작업 내역 관련

* history 저장/조회 Go 파일

확인 함수:

* `ListHistory`
* `GetHistory`
* `GetHistoryStep1`
* `DeleteHistory`

## 앱 셸 / 메뉴 관련

* app shell
* 메뉴 렌더링용 JS/Go 상태 연결부
* audience step 이동 처리부
