 **`support_scriptures`(도움 성구)**를 기준으로, **Step1 → Step2 → temp.json/html → Step3 산출물**까지 흐르도록 보면 됩니다.

아래는 **파일별 체크리스트**입니다.
기준 필드는 우선 아래처럼 두는 것을 전제로 정리하겠습니다.

```json
"support_scriptures": [
  "이사야 40:31",
  "로마서 8:28"
]
```

# 12. 최종 요약

이번 기능은 아래 한 줄로 정리됩니다.

**`support_scriptures`를 Step1에서 추출하고, Step2에서 검토/편집하며, temp.json/html에 반영한 뒤 Step3 산출물까지 자연스럽게 이어지게 한다.**

핵심 파일군만 다시 묶으면:

* Step1 프롬프트/파서
* Step2 state/UI/bind
* temp.json 저장/로드
* html 생성부
* Step3 서비스 점검
* history 재작업 경로

원하시면 다음 단계로는 이 체크리스트를 바탕으로 **“수정 대상 파일명 + 변경 포인트 한 줄 요약표”** 형태로 더 압축해서 정리해드리겠습니다.
