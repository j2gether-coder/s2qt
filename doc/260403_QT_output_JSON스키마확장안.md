화면 구조

핵심 전제는 이미 정해졌습니다.

* 전사와 공통 메타는 **1회**
* QT 생성/편집/산출물은 **연령대별 반복**
* 기준 데이터는 **JSON**
* HTML은 **미리보기**
* PDF/DOCX/PPTX는 **산출물**

그러면 화면도 그 철학을 그대로 따라가야 합니다.

---

# 1. 먼저 화면 구조의 큰 틀

제가 보기엔 QT 작성기는 **2층 구조**로 가는 것이 가장 맞습니다.

## 상위 레벨

**공통 원문 준비 영역**

여기서 한 번만 처리:

* 영상/전사
* 제목
* 본문 성구
* 찬송
* 설교자
* 교회명
* 설교일
* 원본 URL

## 하위 레벨

**연령대별 QT 작업 영역**

여기서 반복 처리:

* 장년
* 청년
* 중고등부
* 어린이

즉 화면 개념은 이렇게 됩니다.

```text id="ui-2level-structure"
[공통 원문 준비]
   └─ 전사 / 메타 / 본문 확정

[연령대별 QT 작업]
   ├─ 장년
   ├─ 청년
   ├─ 중고등부
   └─ 어린이
```

이 구조가 가장 자연스럽습니다.

---

# 2. 기존 Step 1~4를 어떻게 재해석할 것인가

기존 Step 이름을 그대로 쓰면 약간 헷갈릴 수 있습니다.
그래서 저는 이렇게 바꾸는 걸 권합니다.

## 공통 단계

**Step A. 원문 준비**

## 연령대별 단계

**Step 1. 생성 조건 확인**
**Step 2. AI QT 생성**
**Step 3. 내용 검토/편집**
**Step 4. 산출물 생성**

즉 “전사”는 더 이상 Step1이 아니라 **공통 준비 단계**로 분리하는 것이 좋습니다.

# 3. 전체 화면 구성안

가장 추천하는 레이아웃은 아래입니다.

## 화면 상단

공통 원문 준비 카드

## 화면 중단

연령대 선택 탭 또는 카드

## 화면 하단

선택한 연령대의 Step1~4 작업 영역

즉 시각적으로는 이런 느낌입니다.

```text id="screen-layout-overview"
┌─────────────────────────────┐
│ 공통 원문 준비 영역          │
│ 제목 / 본문 / 찬송 / 전사     │
└─────────────────────────────┘

┌─────────────────────────────┐
│ 연령대 선택 탭              │
│ [장년] [청년] [중고등부] [어린이] │
└─────────────────────────────┘

┌─────────────────────────────┐
│ 선택한 연령대 작업 영역       │
│ Step1 생성조건              │
│ Step2 AI 생성              │
│ Step3 검토/편집            │
│ Step4 산출물 생성          │
└─────────────────────────────┘
```

# 4. 공통 원문 준비 화면 설계

이 영역은 **항상 하나만 존재**합니다.

## 4.1 목적

* 한 설교에 대한 공통 원본을 준비하는 곳
* 이후 모든 연령대 작업이 이 데이터를 참조

## 4.2 주요 입력 항목

* 제목
* 본문 성구
* 찬송
* 설교자
* 교회명
* 설교일
* 원본 URL
* 전사문

## 4.3 권장 배치

상단은 메타 입력, 하단은 전사 관련 버튼/상태

예:

```text id="common-source-ui"
[제목] [본문 성구]
[찬송] [설교자]
[교회명] [설교일]
[원본 URL]

[전사 실행] [전사 다시 실행] [전사문 정리]

[전사 상태]
- 다운로드 완료
- 오디오 추출 완료
- Whisper 전사 완료

[전사문 미리보기]
```

## 4.4 저장 결과

이 화면에서 저장되면:

* `sermon_source.json`
* 전사 텍스트
* 공통 메타

가 확정됩니다.

---

# 5. 연령대 선택 화면 설계

이건 탭 또는 카드 형태가 좋습니다.

## 5.1 추천 방식

**탭 방식** 추천

이유:

* 한 설교를 기준으로 여러 audience 버전을 오가야 함
* 장년/청년/중고등부/어린이 비교가 쉬움

예:

* 장년
* 청년
* 중고등부
* 어린이

## 5.2 각 탭의 상태 표시

단순 탭만 두지 말고 상태를 같이 보여주면 좋습니다.

예:

* 장년: 생성 완료
* 청년: 편집 중
* 중고등부: 미생성
* 어린이: 산출물 완료

즉 탭에 작은 배지나 상태 텍스트를 붙이는 것이 좋습니다.

---

# 6. 연령대별 Step1 화면 설계

이건 “자료 입력”이 아니라 **생성 조건 확인** 단계입니다.

## 6.1 목적

* 공통 원문을 바탕으로, 해당 audience용 QT 생성 옵션을 확인

## 6.2 입력/확인 항목

* audience
* template_id
* 문체 옵션(선택)
* 자동 추천 찬송 사용 여부
* 재생성 여부

## 6.3 화면 예시

```text id="audience-step1-ui"
[대상 연령층] 장년
[템플릿] qt_classic
[문체 옵션] 차분함
[추천 찬송 자동 사용] 사용

[생성 요청 정보]
- 제목: ...
- 본문 성구: ...
- 공통 전사 연결됨
```

## 6.4 핵심

이 단계에서는 큰 편집보다
“이 audience로 생성할 준비가 되었는가”를 확인하는 정도면 충분합니다.

---

# 7. 연령대별 Step2 화면 설계

이 단계가 AI 생성 핵심입니다.

## 7.1 목적

* base prompt + audience prompt + request JSON 조합
* LLM 호출
* output JSON 생성

## 7.2 화면 요소

* 생성 버튼
* 재생성 버튼
* 마지막 생성 시각
* 사용된 audience 규칙 표시
* 생성 로그/상태 표시

## 7.3 추천 UI

```text id="audience-step2-ui"
[AI QT 생성] [다시 생성]

[생성 상태]
- 프롬프트 조합 완료
- LLM 요청 중
- JSON 생성 완료

[요약 정보]
- audience: adult
- template_id: qt_classic
- source_id: sermon_001
```

## 7.4 있으면 좋은 기능

* “다른 연령대로 복제 생성”
* “이전 생성본과 비교”
* “생성 JSON 보기”

이건 실무적으로 꽤 유용합니다.

---

# 8. 연령대별 Step3 화면 설계

이 단계가 가장 중요합니다.

기존에는 HTML 편집 중심이었다면,
앞으로는 **JSON 구조 편집 + HTML preview**가 되어야 합니다.

## 8.1 추천 레이아웃

**좌우 2단 구조**를 권장합니다.

* 왼쪽: 구조화 편집
* 오른쪽: HTML preview

```text id="step3-two-pane"
┌──────────────────┬──────────────────┐
│ 구조화 편집 영역   │ HTML 미리보기 영역 │
└──────────────────┴──────────────────┘
```

## 8.2 왼쪽 편집 영역 구성

각 섹션을 카드로 나누는 것이 좋습니다.

### 카드 1. 기본 정보

* 제목
* 본문 성구
* 찬송

### 카드 2. 말씀의 창

* paragraph 1개

### 카드 3. 오늘의 메시지

* 메시지 1 제목
* 메시지 1 본문
* 메시지 2 제목
* 메시지 2 본문
* 메시지 3 제목
* 메시지 3 본문

### 카드 4. 깊은 묵상과 적용

* 항목 1
* 항목 2
* 항목 3

### 카드 5. 오늘의 기도

* paragraph 1개

이렇게 하면 JSON 구조가 깨지지 않습니다.

## 8.3 오른쪽 미리보기 영역

* 선택한 template_id를 적용한 HTML 미리보기
* 스크롤 가능
* 실시간 반영 가능하면 가장 좋음

## 8.4 왜 이렇게 해야 하나

이유는 간단합니다.

* HTML 자유 편집은 구조를 깨뜨릴 수 있음
* QT는 형식 문서라 구조화 편집이 더 적합
* DOCX/PPTX 렌더링도 이 구조가 훨씬 안정적

# 9. 연령대별 Step4 화면 설계

이 단계는 산출물 생성과 관리입니다.

## 9.1 목적

* 최종 output JSON을 바탕으로 산출물 생성
* HTML, PDF, DOCX, PPTX 파일 관리

## 9.2 권장 구성

### 생성 옵션

* 출력 형식 선택
* 템플릿 선택 확인
* 파일명 규칙 확인

### 생성 버튼

* HTML 생성
* PDF 생성
* DOCX 생성
* PPTX 생성
* 전체 생성

### 결과 목록

* HTML 파일 경로
* PDF 파일 경로
* DOCX 파일 경로
* PPTX 파일 경로
* 열기 버튼

## 9.3 화면 예시

```text id="step4-output-ui"
[HTML 생성] [PDF 생성] [DOCX 생성] [PPTX 생성] [전체 생성]

[산출물 상태]
- HTML: 완료
- PDF: 완료
- DOCX: 대기
- PPTX: 대기

[파일 목록]
- html: ...
- pdf: ...
- docx: ...
- pptx: ...
```

# 10. 연령대별 반복 수행 UX

이 구조의 핵심은 **같은 설교 원문으로 여러 audience를 만들 수 있어야 한다**는 점입니다.

그래서 아래 기능이 꼭 있으면 좋습니다.

## 필수 기능

* 장년용 생성
* 청년용 생성
* 중고등부용 생성
* 어린이용 생성

## 있으면 좋은 기능

* “장년용을 기준으로 청년용 다시 생성”
* “다른 연령대 버전 복사 후 수정”
* “연령대별 결과 비교”

즉 audience 탭은 단순 전환이 아니라
**각 audience별 작업공간**이어야 합니다.

# 11. 상태 관리도 이렇게 나누는 게 좋습니다

## 공통 원문 상태

* 원문 미입력
* 전사 완료
* 메타 완료
* 원문 저장 완료

## audience별 상태

* 미생성
* 생성 완료
* 편집 중
* 산출물 일부 완료
* 산출물 전체 완료

이렇게 관리하면 UX가 훨씬 명확해집니다.

# 12. 최종 추천 화면 구조

제가 추천하는 최종 구조는 아래입니다.

## 상단 고정 영역

**공통 원문 준비**

## 중간 탭 영역

**연령대 선택**

* 장년
* 청년
* 중고등부
* 어린이

## 하단 작업 영역

선택한 연령대에 대해:

* Step1 생성 조건 확인
* Step2 AI QT 생성
* Step3 JSON 기반 검토/편집 + HTML preview
* Step4 산출물 생성
