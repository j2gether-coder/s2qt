# 개발환경 구성 정리

## 1. 개요

서비스는 **단독형**과 **구독형**으로 구분하여 개발한다.

- **단독형**: Windows 전용 로컬 실행형
- **구독형**: Linux 기반 서버형 구독 서비스

두 환경은 역할과 운영 방식이 다르므로, 개발환경도 분리해서 설계한다.

---

## 2. 전체 방향

### 단독형
- 대상 OS: **Windows**
- 실행 방식: 로컬 앱
- 저장소: 로컬 SQLite
- UI: 데스크톱 앱
- 특징: 로그인 후 로컬 DB 복호화, 사용자 API Key 사용, 오프라인 또는 로컬 중심 사용

### 구독형
- 대상 OS: **Linux**
- 실행 방식: 웹 서비스
- 저장소: PostgreSQL
- UI: 웹 브라우저 기반
- 특징: 계정/구독/결제/멀티디바이스 VIEW 중심

---

## 3. 단독형 개발환경

### 3.1 운영체제
- **Windows 전용**

초기 단독형은 Windows만 지원하는 것으로 한다.

### 3.2 백엔드/앱 런타임
- **Go**

단독형의 핵심 로직은 Go로 구현한다.

적용 대상:
- 로그인/복호화 로직
- 로컬 DB 처리
- API Key 암호화/복호화
- QT 생성 요청 처리
- 템플릿 관리
- PDF 출력 처리

### 3.3 데스크톱 프레임워크
- **Wails**

Wails를 사용하여 Go 기반 로직과 데스크톱 UI를 연결한다.

적용 이유:
- Go와 결합이 자연스럽다
- Windows 데스크톱 앱 배포에 적합하다
- 로컬 앱 구조에 맞는다

### 3.4 로컬 데이터베이스
- **SQLite**

SQLite를 사용하여 단독형 로컬 데이터를 저장한다.

저장 대상:
- 로컬 계정 메타
- 보안 메타데이터
- 최종 QT 문서
- 성구 reference
- 템플릿 정보
- 유료 템플릿 구매 이력
- 앱 설정

저장하지 않는 것:
- 원본 음성
- 원본 설교문
- 전사 결과
- 구조정리 중간결과
- 평문 API Key

### 3.5 보안 구조
단독형 보안은 아래를 기본으로 한다.

- 로그인 ID + 강한 비밀번호
- 비밀번호 기반 키 유도
- SQLite 암호화
- API Key는 별도 암호화 파일로 관리
- 로그인 후에만 복호화
- 세션 종료 시 메모리 폐기

### 3.6 단독형 UI 방향
- Wails 기반 데스크톱 UI
- 로그인 화면
- QT 생성 화면
- QT 목록/열람 화면
- 템플릿 출력 화면
- 설정 화면

### 3.7 단독형 개발환경 요약
- OS: Windows
- Language: Go
- Desktop Framework: Wails
- DB: SQLite
- 출력: client 측 PDF
- 보안: 로컬 로그인 + DB 암호화 + API Key 암호화 파일

## 4. 구독형 개발환경

### 4.1 운영체제
- **Linux 기반**

구독형 서버는 Linux를 기준으로 운영한다.

적용 이유:
- Go 서버 운영이 단순하다
- PostgreSQL/Nginx/Redis 조합이 안정적이다
- 배포와 자동화가 용이하다
- systemd, cron, 로그 관리에 유리하다

### 4.2 서버 백엔드
- **Go**

구독형 서버의 핵심 API와 비즈니스 로직은 Go로 구현한다.

적용 대상:
- 회원가입/로그인
- 세션/인증 처리
- 구독 상태 확인
- 월 사용량(quota) 관리
- 신규 생성 요청 처리
- 구조정리 및 QT 작성 오케스트레이션
- QT 저장/조회/검색
- 결제/템플릿 구매 처리
- 감사 로그 기록

### 4.3 API 프레임워크
추천 후보:
- Gin
- Echo
- Fiber
- net/http

초기 추천:
- **Gin 또는 Echo**

적용 이유:
- 자료가 많다
- 미들웨어 구성이 편하다
- 인증/REST API 설계가 쉽다
- 초기 서비스 운영에 적합하다

### 4.4 데이터베이스
- **PostgreSQL**

구독형 서버의 기본 DB로 PostgreSQL을 사용한다.

저장 대상:
- 사용자 계정
- 인증/보안 정보
- 구독 상태
- 사용량 정보
- 최종 QT 문서
- 성구 reference
- 템플릿 정보
- 템플릿 구매 이력
- 결제 이력
- 감사 로그

저장하지 않는 것:
- 원본 음성
- 원본 설교문
- 전사 결과
- 구조정리 중간결과
- 임시 PDF

### 4.5 웹 프론트
추천 방향:
- **React + Vite**
- 또는 **Next.js**

초기 추천:
- **React + Vite**

적용 이유:
- 단순하고 가볍다
- 반응형 UI 구성 쉬움
- 모바일/태블릿 VIEW 대응 가능
- eBook 스타일 열람 화면 구현에 적합

확장 시:
- SEO나 페이지 구조가 복잡해지면 Next.js 검토 가능

### 4.6 리버스 프록시 / 웹 서버
- **Nginx** 또는 **Caddy**

초기 추천:
- **Nginx**

역할:
- HTTPS 종단
- reverse proxy
- 정적 파일 서빙
- API 서버 연결

### 4.7 세션 / 캐시 / 큐
선택 요소:
- **Redis**

초기에는 필수는 아니지만, 아래 용도로 유용하다.

활용 가능 영역:
- 세션 저장
- 로그인 실패 횟수 캐시
- rate limiting
- quota 체크 보조
- 작업 큐 상태 관리

초기 MVP:
- PostgreSQL만으로 시작 가능
- 필요 시 Redis 추가

### 4.8 백그라운드 작업
선택 요소:
- **Go worker**

역할:
- 생성 작업 분리
- 메일 발송
- 정기 상태 정리
- 구독 만료/갱신 배치
- 로그 정리

초기에는 API 서버 내 동기 처리 가능하지만,
확장 시 worker 분리를 고려한다.

## 5. 구독형 보안 환경

### 5.1 인증
- 이메일 주소 전체를 로그인 ID로 사용
- 비밀번호는 매번 입력
- 비밀번호 해시 저장
- Argon2id 우선 적용

### 5.2 세션
- 서버 세션 또는 토큰 기반 인증
- HttpOnly / Secure 쿠키 기반 세션 권장
- 로그아웃 시 세션 무효화

### 5.3 저장 원칙
- 최종 QT와 최소 메타정보만 저장
- 원본과 중간결과는 저장하지 않음
- 감사 로그 별도 저장

### 5.4 멀티디바이스
구독형의 핵심 차별점은 다음과 같다.

- PC/모바일 어디서나 접근 가능
- 로그인만 하면 기존 QT 열람 가능
- 서버 저장 기반 VIEW 제공

## 6. 배포 방식

### 6.1 단독형
- Windows 실행 파일 배포
- Wails 패키징
- 로컬 설치형 제공

### 6.2 구독형
Linux 서버 기준으로 아래 중 하나를 사용한다.

#### A. systemd 기반 배포
- Go 바이너리 빌드
- Linux 서버 업로드
- systemd 서비스 등록
- Nginx reverse proxy 구성

장점:
- 단순함
- 초기 운영에 적합

#### B. Docker Compose 기반 배포
- app
- postgres
- redis(선택)
- nginx

장점:
- 환경 재현 용이
- 배포 구성 명확
- 이식성 좋음

초기 추천:
- **systemd + PostgreSQL + Nginx**
- 이후 필요 시 Docker Compose 전환 가능

## 7. 개발환경 비교 표

| 구분 | 단독형 | 구독형 |
|------|--------|--------|
| 운영체제 | Windows | Linux |
| 실행 방식 | 로컬 데스크톱 앱 | 웹 서비스 |
| 핵심 언어 | Go | Go |
| UI/프론트 | Wails | React + Vite 또는 Next.js |
| DB | SQLite | PostgreSQL |
| 보안 저장 | 로컬 암호화 DB + 암호화 파일 | 서버 인증/세션/보안 모델 |
| API Key | 사용자 제공, 로컬 암호화 관리 | 서버 운영자 관리 |
| 생성 처리 | client 중심 | server 중심 |
| 열람 방식 | 로컬 열람 | 멀티디바이스 VIEW |
| 배포 | Windows 앱 설치 | Linux 서버 배포 |
| 출력 | client PDF | VIEW 중심, PDF 보조 |

## 8. 최종 권장 조합

### 단독형
- Windows
- Go
- Wails
- SQLite

### 구독형
- Linux
- Go + Gin(Echo 가능)
- PostgreSQL
- React + Vite
- Nginx
- Redis는 선택
- Go worker는 선택 또는 확장 단계 적용

## 9. 한 줄 정리

**단독형은 Windows 기반의 Go + Wails + SQLite 구조로, 구독형은 Linux 기반의 Go + PostgreSQL + 웹 프론트 구조로 분리하여 개발한다.**
