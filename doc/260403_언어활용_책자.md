# 📑 JSON 기반 문서 생성 언어 및 아키텍처 정리

## 1. 언어별 장단점

| 언어 | 강점 | 약점 | 적합한 산출물 |
|------|------|------|---------------|
| **Python** | - 풍부한 라이브러리 (ReportLab, python-docx, python-pptx, WeasyPrint)<br>- JSON 처리 용이<br>- 커뮤니티 지원 강력 | - 실행 속도는 Go보다 느림 | PDF, Word, PPT, HTML |
| **JavaScript/Node.js** | - Docxtemplater 등 템플릿 기반 라이브러리<br>- 웹과 자연스러운 연동<br>- JSON 직렬화/역직렬화 최적 | - PDF 생성은 브라우저 기반 렌더링 필요 (Puppeteer 등) | Word, PPT, HTML, PDF(웹 기반) |
| **Go** | - 빠른 성능, 배포 용이<br>- Wails로 GUI/웹앱 제작 가능 | - 문서 생성 라이브러리 부족<br>- PDF/Word/PPT 지원 제한적 | HTML, PDF(기본 수준) |
| **Java** | - Apache POI 등 안정적인 라이브러리<br>- 대규모 엔터프라이즈 환경 적합 | - 상대적으로 무겁고 생산성 낮음 | Word, PPT, Excel |
| **C#/.NET** | - OfficeInterop, OpenXML SDK 등 강력한 지원<br>- Windows 환경 최적 | - 크로스플랫폼 제약 | Word, PPT, Excel |

---

## 2. 인쇄 시나리오별 고려사항

### 📄 한 장 인쇄
- **HTML → PDF 변환**  
  - Python: `WeasyPrint`, `pdfkit`  
  - JS: `Puppeteer`, `wkhtmltopdf`  
- 장점: 빠른 렌더링, GUI에서 미리보기 가능  

### 📚 책자 인쇄
- **Python ReportLab**: 페이지 번호, 목차, 이미지 삽입 등 세밀한 제어 가능  
- **LaTeX 연동**: JSON → LaTeX 템플릿 → PDF 변환 → 고품질 책자 인쇄에 최적  
- 장점: 전문적인 레이아웃, 인쇄소 수준의 품질 확보  

---

## 3. 추천 아키텍처 (Go + Wails + Python + JS)

1. **Go(Wails)**  
   - 사용자 인터페이스, JSON 입력/편집, 인쇄 옵션 선택  
   - 배포 및 성능 담당  

2. **Python**  
   - 문서 생성 엔진 (단순 출력은 WeasyPrint, 책자 출력은 ReportLab/LaTeX)  

3. **JavaScript(Webview)**  
   - HTML 미리보기 및 PDF 변환  

---

## 4. 최종 전략
- **단순 출력**: HTML 템플릿 → PDF 변환 (빠른 미리보기)  
- **책자 출력**: Python ReportLab/LaTeX 기반 PDF 생성 (고품질 인쇄)  
- **최종 결과물**: 항상 PDF로 통일 → 인쇄 친화적  

---

👉 결론: **GUI는 Go(Wails)**로, **문서 생성은 Python**으로 맡기고, **미리보기는 JS(Webview)**로 처리하는 구조가 가장 안정적이며, 최종 결과물은 **PDF 중심**으로 가져가는 것이 적합합니다.
