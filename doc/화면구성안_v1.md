# QT 문서 생성 프로그램 화면 구성안 (개정안)

## 1. 문서 목적

본 문서는 QT 문서 생성 프로그램의 **화면 구조 및 사용자 흐름을 정의하기 위한 설계 문서**이다.

본 프로그램은 전사 기능을 포함하는 도구가 아니라,
**이미 확보된 전사 데이터를 기반으로 QT 문서를 생성·편집·저장·출력하는 것을 목적**으로 한다.

---

## 2. 설계 기본 방향

### 2.1 핵심 원칙

* UI는 **기능 나열형이 아닌 단계형(Task Flow 기반)**으로 구성한다.
* 사용자는 자연스럽게 **순차 흐름(Input → Generate → Edit → Export)**을 따르도록 유도한다.
* 문서 구조는 고정하며, 사용자는 **본문 내용만 수정**할 수 있도록 제한한다.
* 상태(State)는 단계별로 분리하여 관리한다.
* 향후 확장을 고려하여 **작업 유형 기반 구조**로 설계한다.

---

### 2.2 기존 구조의 문제점

현재 단일 화면 구조는 다음 문제를 가진다.

* 기능 과밀 (입력 / 생성 / 편집 / 출력 혼재)
* 상태 관리 복잡도 증가
* 이벤트 의존성 증가
* 유지보수 어려움 (`main.js` 비대화)
* 기능 확장 시 UI 영향 범위 확대

→ 해결 방안: **단계별 화면 분리**

---

## 3. 전체 흐름 구조

### 3.1 작업 단계

1. 자료 입력
2. 초안 생성
3. 본문 편집
4. 저장 / 출력

---
## 3.1 1단계. 자료 입력

### 목적
QT 문서 생성을 위한 원본 자료를 준비하고, 이후 문서 생성에 필요한 기본 메타정보를 입력 또는 보정한다.

### 작업 흐름
1. 사용자는 먼저 입력 유형(동영상, 음성, 텍스트)에 따라 원본 자료 작업을 수행한다.
2. 원본 자료 또는 전사 결과가 준비되면, 제목, 본문 성구, 설교자, 교회명 등의 메타정보를 입력하거나 수정한다.

### 세부 절차
#### (1) 원본 자료 작업
- 동영상: URL 입력 → 메타정보 불러오기 → 다운로드/전사 실행
- 음성: 원본 정보 입력 → 전사 실행
- 텍스트: 원문 직접 입력

#### (2) 메타정보 입력/보정
- 제목(필수)
- 본문 성구(필수)
- 설교자(선택)
- 교회명(선택)

### 비고
- 입력 유형에 따라 원본 처리 절차가 달라질 수 있다.
- 동영상의 경우 메타정보 자동 추출 결과를 바탕으로 사용자가 수정할 수 있다.
- 본 단계의 핵심은 메타정보 선입력이 아니라, 원본 자료 준비를 우선하는 것이다.

### 3.2 상태 흐름 (중요)

각 단계는 다음 상태를 생성/소비한다.

| 단계 | 입력                    | 출력                    |
| -- | --------------------- | --------------------- |
| 1  | 사용자 입력                | metadata + transcript |
| 2  | metadata + transcript | draft (HTML + prompt) |
| 3  | draft                 | edited content        |
| 4  | edited content        | final HTML + PDF      |

→ **단계 간 데이터 계약(Data Contract) 명확화 필요**

## 4. 단계별 화면 설계

---

## 4.1 1단계. 자료 입력

### 목적

QT 생성에 필요한 **기초 데이터 확보**

### 입력 데이터

* 제목
* 본문 성구
* 설교자
* 교회명
* 설교일
* 입력 유형 (video / audio / text)
* URL 또는 식별값
* 전사문

### 주요 기능

* 입력 폼
* 전사문 붙여넣기 / 불러오기
* 임시 저장

### UI 구성

* 입력 폼 영역
* 작업 시작 버튼
* 임시 저장 상태 표시
* 최근 작업 불러오기 (추후)

### 설계 포인트

* 전사 기능은 제외 (입력 데이터로 취급)
* validation 필요 (필수값 체크)

---

## 4.2 2단계. 초안 생성

### 목적

입력 데이터를 기반으로 **QT 초안 생성**

### 주요 기능

* 프롬프트 생성
* 프롬프트 확인
* 초안 생성 (LLM 호출)
* 초안 미리보기

### UI 구성

* 프롬프트 생성 버튼
* 프롬프트 표시 영역
* 초안 생성 버튼
* HTML 미리보기

### 설계 포인트

* 프롬프트는 디버깅/튜닝을 위해 노출
* 초안 결과는 **immutable draft 상태로 관리**

---

## 4.3 3단계. 본문 편집

### 목적

초안을 사람이 검토하여 **최종 문서로 보정**

### 편집 대상

* 제목
* 본문 요약
* 오늘의 메시지
* 깊은 묵상과 적용
* 오늘의 기도

### UI 구성

* 섹션별 편집 입력창
* 미리보기 패널
* 저장 버튼

### 편집 방식 (권장)

✔ 섹션별 편집 방식

### 설계 포인트

* 레이아웃 수정 불가
* HTML 직접 수정 제한
* 구조 유지 강제

---

### 문서 구조 (고정)

* 제목
* 본문/찬송
* 말씀의 창 (요약)
* 오늘의 메시지
* 깊은 묵상과 적용
* 오늘의 기도
* Footer

---

## 4.4 4단계. 저장 / 출력

### 목적

최종 결과를 **파일 및 DB로 저장하고 출력**

### 주요 기능

* 작업 저장
* HTML 저장
* PDF 생성
* 기존 작업 열기

### UI 구성

* 저장 버튼
* HTML 다운로드
* PDF 생성 버튼
* 파일 상태 표시

### 설계 포인트

* 출력은 항상 **편집 완료 이후만 허용**
* PDF 생성은 HTML 기반으로 수행

## 10. 결론

본 시스템은 다음과 같이 정의된다.

> **전사 데이터를 입력으로 받아 QT 문서를 생성하고, 사람이 보정한 후 저장 및 출력하는 문서 생성 시스템**

* 사용자 흐름 명확화
* 유지보수성 향상

* 데이터 구조 안정성 확보
