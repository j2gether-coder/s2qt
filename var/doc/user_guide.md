# 사용 가이드

## 프로그램 개요

S2QT는 설교를 바탕으로 QT 문서를 작성할 수 있도록 돕는 도구입니다.  
자료 준비, AI(LLM) 초안 생성, 검토 및 편집, 문서 생성의 순서로 사용할 수 있습니다.

기본 흐름은 다음과 같습니다.

1. QT 준비
2. Step1. AI(LLM) 이용
3. Step2. 검토 및 편집
4. Step3. QT 문서 생성

## 1. QT 준비

QT 준비에서는 문서 생성을 위한 원본 자료와 기본 정보를 준비합니다.

지원하는 자료 입력 방식은 다음과 같습니다.

- 동영상
- 오디오
- 텍스트

자료 준비가 완료되면 전사 또는 입력 내용이 정리되고, 이후 Step1부터 작업을 이어갈 수 있습니다.

### 기본 정보 입력

자료 준비 단계에서는 다음과 같은 기본 정보를 함께 확인하거나 입력할 수 있습니다.

- 제목
- 본문 성구
- 찬송
- 설교자
- 교회명
- 설교일

기본 정보는 이후 Step2, Step3 및 작업 내역에서도 함께 사용됩니다.

## 2. Step1. AI(LLM) 이용

Step1에서는 AI(LLM)를 이용하여 QT 초안을 생성합니다.

### 주요 흐름

1. 프롬프트 생성
2. AI 결과 확인 또는 붙여넣기
3. 결과 저장

### AI 사용 방식

환경설정의 부가 기능에서 AI 사용 방식을 선택할 수 있습니다.

- 수동: 프롬프트를 복사하여 외부 AI 서비스에서 결과를 받아 붙여넣는 방식
- 원격: 설정된 API KEY를 이용하여 원격 AI 기능을 사용하는 방식
- 로컬: 추후 확장 예정

### 결과 저장

AI 결과를 확인한 후 저장하면 Step2에서 이어서 검토 및 편집할 수 있습니다.

## 3. Step2. 검토 및 편집

Step2에서는 Step1에서 생성한 초안을 검토하고 수정합니다.

### 편집 대상 예시

- 말씀의 창
- 오늘의 메시지
- 깊은 묵상과 적용
- 오늘의 기도

필요한 내용을 직접 수정한 뒤 저장할 수 있으며, 미리보기 기능을 통해 결과를 확인할 수 있습니다.

### 참고

- 제목과 본문 성구는 읽기 전용 정보로 함께 표시될 수 있습니다.
- Step2에서 저장한 내용은 Step3 문서 생성에 반영됩니다.

## 4. Step3. QT 문서 생성

Step3에서는 QT 결과물을 실제 문서 형태로 생성합니다.

### 생성 가능한 형식

현재 기본적으로 다음 형식을 사용할 수 있습니다.

- PDF
- PNG

일부 형식은 버전 또는 설정에 따라 추후 확장될 수 있습니다.

### 기본 사용 순서

1. 생성할 결과 형식 선택
2. 실행
3. 생성 결과 확인
4. 파일 저장

생성된 결과물은 화면에서 확인 후 저장할 수 있습니다.

## 5. 작업 내역

작업 내역에서는 이전에 저장된 작업을 다시 확인하고 재작업할 수 있습니다.

### 주요 기능

- 검색
- 정렬
- 재작업
- 삭제

### 기본 사용 방식

작업 내역 화면에서는 제목, 본문 성구, 저장일, 연령대 등을 기준으로 작업을 확인할 수 있습니다.  
필요한 작업을 선택한 뒤 다시 작업을 이어갈 수 있습니다.

### 재작업

재작업은 저장된 기본 정보와 Step1 결과를 기준으로 다시 이어서 작업하는 기능입니다.  
일반적으로 1건 선택 기준으로 사용하는 것을 권장합니다.

## 6. 환경설정

환경설정에서는 문서 작성과 부가 기능 사용을 위한 기본 값을 설정할 수 있습니다.

### 6-1. 기본 정보

기본 정보에서는 다음 항목을 설정할 수 있습니다.

- 기본 이메일
- PIN 설정
- 교회/브랜드 설정
- 기본 하단 문구

기본 이메일과 PIN은 저장된 민감정보 보호에 사용됩니다.

#### 교회/브랜드 로고 설정 안내

교회/브랜드 설정에서는 산출물 하단에 사용할 로고 또는 브랜드 이미지를 등록할 수 있습니다.

로고 입력 시에는 다음 사항을 권장합니다.

- 밝은 배경에서도 글자와 심볼이 또렷하게 보이는 이미지를 사용합니다.
- 흰색 또는 매우 연한 색의 글자가 포함된 로고는 하단에서 잘 보이지 않을 수 있습니다.
- 너무 복잡하거나 글자가 작은 이미지는 축소 시 식별이 어려울 수 있습니다.
- 가능한 경우 배경이 투명한 PNG 형식을 권장합니다.

로고 미리보기는 흰색 배경 기준으로 확인하는 것을 권장합니다.  
미리보기에서 글자나 심볼이 잘 보이지 않는 경우, 다른 로고 또는 브랜드 이미지를 사용하는 것이 좋습니다.
※ 로고는 흰색 배경 기준 미리보기에서 글자와 심볼이 분명하게 보이는 이미지를 권장합니다.

교회명 입력은 한 줄로 입력하되, 권장 형식은 다음과 같습니다.

- 교단명, 교회명

예시:

- 대한예수교장로회, 은혜교회
- 기독교대한감리회, 소망교회

### 6-2. 부가 기능

부가 기능에서는 다음 항목을 설정할 수 있습니다.

- AI 사용 방식
- 원격 AI API KEY
- SMTP 사용 여부
- SMTP 서버 정보
- SMTP 앱 비밀번호

#### 민감정보 관리 안내

원격 AI API KEY, SMTP 앱 비밀번호, 이메일 계정 정보 등은 민감정보에 해당합니다.

예시:

- 원격 AI API KEY
- SMTP 앱 비밀번호
- 이메일 계정 정보
- 외부 서비스 연동에 사용하는 인증키 또는 토큰

이 정보는 외부에 공유하거나 문서, 화면 캡처, Git 저장소 등에 포함하지 않도록 주의해야 합니다.
공용 PC 또는 여러 사용자가 함께 사용하는 환경에서는 입력값 저장 여부와 PIN 설정을 반드시 확인하는 것을 권장합니다.

### 6-3. 메뉴 설정

메뉴 설정에서는 사이드 메뉴의 표시 여부와 표시명을 설정할 수 있습니다.

예를 들어 내부 분류는 유지한 상태에서 다음과 같이 표시명을 바꿔 사용할 수 있습니다.

- 중고등부 QT → 학생부 QT
- 청년 QT → 청년부 QT

메뉴 설정은 화면 표시용이며, 내부 연령대 구분 값은 그대로 유지됩니다.

### 6-4. 사용 안내

사용 안내에서는 라이선스와 사용 가이드를 확인할 수 있습니다.

## 7. 사용 시 참고 사항

- PIN은 민감정보 보호를 위한 보안 수단으로 사용됩니다.
- 발신 이메일은 기본 정보에 저장된 이메일을 기준으로 사용합니다.
- 기본 하단 문구는 짧게 작성하는 것을 권장합니다.
- 일부 기능은 추후 버전에서 보완되거나 확장될 수 있습니다.

## 8. 자주 확인할 사항

다음 항목은 작업 전에 한 번 더 확인하는 것을 권장합니다.

- 제목과 본문 성구가 올바른지
- AI 결과가 의도에 맞는지
- 문서 생성 형식이 올바르게 선택되었는지
- 외부 전송 전에 최종 문서를 검토했는지
