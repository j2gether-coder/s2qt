# S2QT 템플릿 사용 가이드

## 1. 개요

S2QT의 템플릿 기능은 Step3에서 생성되는 **PDF / PNG 산출물**에 공통 디자인을 적용하기 위한 기능입니다.

본 기능은 다음 원칙을 따릅니다.

- Step1 / Step2 / Step3의 기존 흐름은 유지합니다.
- 템플릿은 **환경설정 > 템플릿**에서 선택합니다.
- 실제 적용은 **Step3 산출물 생성 후 후처리 방식**으로 수행됩니다.
- 템플릿은 기본 산출물을 대체하지 않고, **생성된 결과물 위에 적용**됩니다.

---

## 2. 적용 대상

템플릿은 다음 산출물에 적용됩니다.

- PDF
- PNG

즉, 템플릿을 사용하면 Step3에서 생성된 `temp.pdf`, `temp.png`에 동일한 템플릿 디자인이 반영됩니다.

---

## 3. 기본 동작 방식

템플릿 기능의 전체 흐름은 다음과 같습니다.

1. 사용자가 환경설정에서 템플릿 사용 여부를 설정합니다.
2. 사용자가 템플릿 목록에서 템플릿을 선택합니다.
3. Step3에서 PDF / PNG를 생성합니다.
4. 선택된 템플릿이 후처리 방식으로 PDF / PNG에 적용됩니다.

정리하면:

- **선택**은 환경설정에서
- **적용**은 Step3에서
- **실제 적용 시점**은 산출물 생성 이후입니다.

---

## 4. 템플릿 폴더 구조

템플릿은 `var/template` 아래에 위치합니다.

### 4.1 공통 구조

```text
var/
  template/
    no_image.png
    tpl_0001/
      manifest.json
      template.png
````

### 4.2 파일 설명

#### `no_image.png`

* 템플릿 미리보기가 없을 때 사용하는 기본 이미지입니다.
* 설정 화면의 미리보기 fallback 이미지로 사용됩니다.

#### `tpl_0001`

* 템플릿 1건을 의미하는 폴더입니다.
* 템플릿별로 고유 폴더를 가집니다.

#### `manifest.json`

* 템플릿의 메타 정보 파일입니다.
* 템플릿명, 분류, 사용 여부, 이미지 파일명, PNG 배치 정보 등을 정의합니다.

#### `template.png`

* 템플릿의 공통 원본 이미지입니다.
* 다음 용도로 공통 사용됩니다.

  * 설정 화면 미리보기용 썸네일 원본
  * PDF 배경용 원본
  * PNG 배경용 원본

---

## 5. 템플릿 ID 및 폴더명 규칙

템플릿 폴더명은 다음과 같은 고정 ID 형태를 권장합니다.

* `tpl_0001`
* `tpl_0002`
* `tpl_0003`

권장 이유:

* 정렬이 깔끔합니다.
* `app_settings`에 저장되는 `selected_id`와 일관성이 좋습니다.
* 날짜형 이름보다 운영 시 식별이 안정적입니다.

권장 예:

```text
tpl_0001
```

비권장 예:

```text
tpl_260420_01
```

---

## 6. manifest.json 작성 기준

현재 권장하는 최소 manifest 구조는 아래와 같습니다.

```json
{
  "id": "tpl_0001",
  "name": "테스트 템플릿 001",
  "category": "monthly",
  "enabled": true,
  "template_image": "template.png",
  "png_placement": {
    "left_px": 112,
    "top_px": 124,
    "width_px": 2254,
    "height_px": 3254,
    "fit_mode": "contain",
    "align_x": "center",
    "align_y": "top",
    "debug_rect": false
  }
}
```

### 항목 설명

#### `id`

* 템플릿 고유 ID입니다.
* 보통 폴더명과 동일하게 맞춥니다.

예:

```json
"id": "tpl_0001"
```

#### `name`

* 설정 화면 목록에 표시되는 템플릿명입니다.

#### `category`

* 템플릿 분류입니다.
* 아래 값 중 하나를 사용합니다.

  * `monthly`
  * `seasonal`
  * `liturgical`

참고:

* `all`은 저장값이 아니라 **화면 필터용**입니다.

#### `enabled`

* 템플릿 사용 가능 여부입니다.
* `true`이면 목록에 표시 및 선택 가능
* `false`이면 사용하지 않음

#### `template_image`

* 템플릿 공통 이미지 파일명입니다.
* 현재 구조에서는 `template.png` 하나로 공통 처리합니다.

#### `png_placement`

* PNG 합성 시 `temp.png`를 어떤 위치와 크기로 배치할지 정의합니다.

---

## 7. PNG 배치 옵션 설명

`png_placement`는 PNG 후처리 적용 시 중요합니다.

### 예시

```json
"png_placement": {
  "left_px": 112,
  "top_px": 124,
  "width_px": 2254,
  "height_px": 3254,
  "fit_mode": "contain",
  "align_x": "center",
  "align_y": "top",
  "debug_rect": false
}
```

### 항목 설명

#### `left_px`

* 배치 시작 X 좌표

#### `top_px`

* 배치 시작 Y 좌표

#### `width_px`

* 배치 영역 너비

#### `height_px`

* 배치 영역 높이

#### `fit_mode`

* 이미지 맞춤 방식
* 사용 값:

  * `contain`
  * `cover`

#### `align_x`

* 가로 정렬 방식
* 사용 값:

  * `left`
  * `center`
  * `right`

#### `align_y`

* 세로 정렬 방식
* 사용 값:

  * `top`
  * `center`
  * `bottom`

#### `debug_rect`

* 디버그용 사각형 표시 여부
* 보통 운영에서는 `false`

## 8. 환경설정에서 템플릿 선택 방법

### 8.1 진입 경로

환경설정 > 템플릿

### 8.2 화면 구성

템플릿 설정 화면은 다음 영역으로 구성됩니다.

#### 템플릿 설정

* 템플릿 사용 체크박스

#### 템플릿 선택

* 분류 라디오

  * 전체
  * 월별
  * 계절별
  * 절기별
* 템플릿 목록
* 새로 고침 버튼
* 미리보기 영역

### 8.3 사용 절차

1. `템플릿 사용` 체크
2. 분류 선택
3. 목록에서 템플릿 선택
4. 미리보기 확인
5. Step3 실행

---

## 9. 템플릿 목록 새로 고침

새 템플릿을 `var/template` 아래에 추가한 뒤에는 목록 새로 고침이 필요합니다.

### 새로 고침이 필요한 경우

* 새 템플릿 폴더를 추가한 경우
* manifest.json을 수정한 경우
* template.png를 교체한 경우

### 새로 고침 후 확인 항목

* 목록에 템플릿이 보이는지
* 분류 필터에서 정상 표시되는지
* 선택 시 미리보기가 나오는지

---

## 10. Step3에서 템플릿 적용

Step3에서는 다음 순서로 동작합니다.

1. HTML / PDF / PNG 기본 산출물 생성
2. 템플릿 사용 여부 확인
3. 선택된 템플릿 확인
4. PDF / PNG에 템플릿 후처리 적용

즉, 템플릿은 Step3의 **기본 생성 로직을 바꾸지 않고**, 후처리로만 적용됩니다.

---

## 11. PDF 적용 방식

PDF 템플릿은 다음 방식으로 적용됩니다.

* Step3에서 생성된 콘텐츠 기준으로 PDF를 구성
* 템플릿 이미지를 배경 레이어처럼 적용
* 본문 내용은 그 위에 유지

핵심:

* 원본 콘텐츠는 유지
* 배경 디자인만 템플릿으로 입힘

---

## 12. PNG 적용 방식

PNG 템플릿은 다음 방식으로 적용됩니다.

* `template.png`를 배경으로 사용
* Step3에서 생성된 `temp.png`를 전경으로 사용
* `png_placement` 기준으로 위치/크기/정렬을 적용

핵심:

* PNG는 배경 + foreground 합성 방식입니다.

---

## 13. 미리보기 동작 방식

미리보기는 선택된 템플릿 1건에 대해서만 로드됩니다.

### 원칙

* 목록에서는 이미지 스캔/썸네일 생성을 하지 않음
* 선택된 템플릿만 미리보기 처리
* 미리보기용 썸네일이 없으면 생성 가능
* 실패하면 `no_image.png` 사용

이 구조는 불필요한 전체 이미지 스캔과 메모리 사용을 줄이기 위한 것입니다.

---

## 14. 운영상 주의사항

### 14.1 템플릿 스캔 범위

템플릿 스캔은 반드시 `var/template` 직하 디렉터리만 대상으로 합니다.

즉:

* `var/template/tpl_0001`
* `var/template/tpl_0002`

까지만 보고, 하위 전체 재귀 스캔은 하지 않습니다.

### 14.2 목록 조회 시 이미지 처리 금지

목록 조회 단계에서는:

* 이미지 디코드
* 썸네일 생성
* 전체 PNG 스캔

을 하지 않는 것이 원칙입니다.

### 14.3 실제 미리보기는 선택 시점에만

미리보기는 선택한 템플릿 1건만 처리해야 합니다.

---

## 15. 템플릿 추가 절차

새 템플릿을 추가할 때는 다음 절차를 따릅니다.

### 절차

1. `var/template` 아래 새 폴더 생성
2. 폴더명은 `tpl_0001` 형식 사용
3. `manifest.json` 작성
4. `template.png` 배치
5. 앱에서 템플릿 화면 진입
6. 새로 고침 실행
7. 목록/미리보기 확인
8. Step3 적용 테스트

### 예시

```text
var/template/tpl_0002/
  manifest.json
  template.png
```

---

## 16. 테스트 체크리스트

### 설정 화면

* [ ] 템플릿 목록에 신규 템플릿이 보인다
* [ ] 분류 라디오로 필터링된다
* [ ] 템플릿 사용 체크가 정상 동작한다
* [ ] 선택 라디오가 정상 동작한다
* [ ] 선택 시 미리보기가 표시된다

### Step3

* [ ] PDF에 템플릿이 적용된다
* [ ] PNG에 템플릿이 적용된다
* [ ] 템플릿 사용 해제 시 적용되지 않는다

### 예외

* [ ] 템플릿이 없으면 no_image가 표시된다
* [ ] manifest 오류 시 목록에 비정상 노출되지 않는다
* [ ] template.png가 없으면 미리보기 fallback이 동작한다

---

## 17. 자주 발생하는 문제

### 문제 1. 템플릿이 목록에 보이지 않음

확인 항목:

* 폴더가 `var/template` 아래에 있는지
* 폴더명이 비어 있지 않은지
* `manifest.json`이 있는지
* `enabled`가 `true`인지

### 문제 2. 미리보기가 보이지 않음

확인 항목:

* `template.png` 존재 여부
* `template_image` 값이 올바른지
* `no_image.png` 존재 여부

### 문제 3. Step3에 적용되지 않음

확인 항목:

* `템플릿 사용` 체크 여부
* 템플릿 선택 여부
* `selected_id` 저장 여부
* Step3 실행 시 후처리 호출 여부

### 문제 4. 메모리 사용량이 큼

확인 항목:

* 목록 조회 시 전체 이미지 스캔을 하고 있지 않은지
* 선택된 템플릿 1건만 미리보기 처리하는지
* 원본 `template.png`가 너무 큰지

---

## 18. 권장 운영 원칙

현재 단계에서는 다음 원칙을 권장합니다.

* 템플릿 1건 = 1폴더
* 폴더명 = 템플릿 ID
* 공통 이미지 = `template.png`
* 설정값은 `app_settings` 저장
* 목록은 텍스트 중심
* 미리보기는 선택된 템플릿만
* PDF / PNG 적용은 후처리로만 수행

---

## 19. 권장 최소 구조 요약

```text
var/
  template/
    no_image.png
    tpl_0001/
      manifest.json
      template.png
```

이 구조만 갖추면 기본 테스트를 진행할 수 있습니다.

---

## 20. 결론

S2QT 템플릿 기능은 다음과 같이 이해하면 가장 쉽습니다.

* 템플릿은 Step3 결과물에 입히는 디자인 레이어
* 선택은 환경설정에서
* 적용은 Step3에서
* 구조는 `manifest.json + template.png`
* 스캔은 `var/template` 직하만
* 미리보기는 선택된 템플릿 1건만

이 기준을 따르면 템플릿 추가, 테스트, 운영 반영을 안정적으로 진행할 수 있습니다.
